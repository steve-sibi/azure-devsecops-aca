# Security Policy

## Reporting a vulnerability

If you believe you’ve found a security vulnerability, please do not open a public issue.

- Preferred: open a private GitHub Security Advisory for this repository.
- If that’s not available, contact the maintainer directly.

Please include:
- A clear description of the issue and impact
- Reproduction steps (proof-of-concept if possible)
- Affected files/paths and any relevant logs

## Safe use disclaimer

This project downloads and scans remote content. Only scan URLs you own or are authorized to test. The authors are not responsible for misuse.
